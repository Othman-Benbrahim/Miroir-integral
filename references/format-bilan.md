# Format Bilan — Miroir Intégral

> Tous les formats de sortie du bilan, par partie.
> Utiliser pendant l'ÉTAPE 3 pour structurer les outputs.

---

## Format universel (s'applique à chaque hypothèse)

```
Observation  : [formulation exacte ou comportement observable]
Hypothèse    : [interprétation — vivante, non académique]
Contre-signe : [ce qui nuance ou limite cette hypothèse]
Pas concret  : [action d'observation dans les 7 prochains jours]
```

---

## PARTIE 1 — Diagnostic de surface

### Profil Big Five (bref, bienveillant)

```
[Dimension] — [Niveau estimé %] :
[Description en 2 lignes : ce que ça apporte + ce que ça coûte en l'état]
```

Limiter à 2–3 dimensions, celles les plus saillantes dans la conversation.

### EMS dominant

```
Schéma actif : [nom lisible — ex : "Peur d'être abandonné" plutôt que "Abandon"]
Domaine      : [Déconnexion / Autonomie / Limites / Orientation / Hypervigilance]
Observation  : [...]
Hypothèse    : [...]
Contre-signe : [...]
Pas concret  : [...]
```

Si deux schémas actifs :
```
Schéma secondaire : [nom]
Lien au schéma principal : [comment ils se renforcent mutuellement]
```

### Position Kegan

```
Position estimée : [Niveau 3 — Socialisé / Transition 3→4 / Niveau 4 — Auto-Auteur]
Ce que ça signifie ici : [une phrase concrète sur le coût ou l'enjeu]
Ce qui est encore Sujet : [ce qui gère l'utilisateur sans qu'il puisse le voir]
Ce qui devient Objet  : [ce qu'il commence à tenir à distance]
```

---

## PARTIE 2 — Cartographie des parties (IFS)

### Par partie identifiée

```
Partie   : [Manager / Pompier / Exilé] — [nom évocateur si possible]
Comment elle se manifeste : [...]
Intention positive cachée : [ce qu'elle protège — jamais "c'est le problème"]
Besoin non satisfait      : [ce dont elle aurait besoin]
```

### Énergie Self (si présente)

```
Moments de Self : [quand et comment l'utilisateur s'est observé avec recul]
Ressource       : [comment cette présence peut être amplifiée]
```

### Dynamique de parties (si conflit visible)

```
Tension principale : [Manager X protège l'Exilé Y de la peur Z]
Ce que ça produit : [le comportement ou blocage observé]
```

---

## PARTIE 3 — Récit central

```
Séquence dominante : [contamination / rédemption / mixte]
Trame              : [le pattern en 2 phrases — sans jargon]
Imago actif        : [personnage internalisé + injonction associée, si identifiable]
Tournant narratif  : [le moment où une bifurcation est possible]
```

### Si séquence de contamination dominante

```
Pattern : "chaque fois que j'avance vers [X], [Y] vient tout défaire"
Origine probable : [lien avec EMS ou Imago]
Bifurcation disponible : [...]
```

### Si séquence de rédemption présente

```
Ressource narrative : [la capacité à trouver du sens — la nommer explicitement]
Comment l'amplifier : [...]
```

---

## PARTIE 4 — Ombre et Archétype (MODE COMPLET)

```
Ombre probable   : [trait ou qualité niés — formulation douce, hypothèse]
Signal observé   : [projection ou réaction spécifique]
Archétype actif  : [nom]
Pattern          : [comment il se manifeste dans les réponses]
Force disponible : [la puissance de cet archétype hors mode peur]
Contre-signe     : [ce qui nuance — toujours présent pour cette couche]
```

---

## PARTIE 5 — Protocole de transformation

### Format standard (2 interventions max)

```
Intervention 1 : [nom]
Fondement      : [IFS / ACT / Ecker — une phrase]
Pourquoi ici   : [lien explicite avec ce qui a émergé]
Instructions   :
  1. ...
  2. ...
  3. ...
Indicateur     : [comment savoir que ça travaille]
Durée          : [...] / Fréquence : [...]
```

### Format condensé (si session courte)

```
Priorité d'action : [une seule intervention, la plus adaptée]
Pas minimal       : [action concrète dans les 48h]
Point d'observation : [dans 7 jours]
```
